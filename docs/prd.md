### **H Wallet 用户注册功能 PRD**

**目标**：设计一个安全、流畅的邮箱注册流程，用户验证邮箱后，系统自动为其创建 OKX Agent Wallet，并引导用户完善基础资料。

---

#### **1. 功能需求列表**

| 编号 | 描述 | 优先级 |
| :--- | :--- | :--- |
| FR-01 | **邮箱注册页面**：用户输入邮箱、密码（需二次确认）、同意服务协议，点击获取验证码。 | P0 |
| FR-02 | **邮箱验证码发送**：点击“获取验证码”后，后端向用户邮箱发送6位数字验证码，前端按钮进入60秒倒计时。 | P0 |
| FR-03 | **验证码校验与注册**：用户输入收到的验证码，点击“注册”按钮，后端校验验证码有效性。 | P0 |
| FR-04 | **自动创建钱包**：邮箱验证通过后，后端系统自动调用 OKX Agent SDK，为该用户创建一个新的 OKX Agent Wallet，并将钱包地址与用户账户绑定。 | P0 |
| FR-05 | **创建用户资料**：注册成功时，系统使用邮箱前缀作为默认昵称，创建一个基础的个人资料记录。 | P0 |

| FR-07 | **输入校验与提示**：前端对邮箱格式、密码强度（至少8位，含字母和数字）、验证码格式进行实时校验并给出友好提示。 | P1 |
| FR-08 | **风控策略**：同一邮箱60秒内只能请求一次验证码；同一IP地址24小时内最多请求10次验证码；验证码有效期为5分钟。 | P1 |
| FR-09 | **异常处理**：处理验证码错误、过期、邮箱已注册、网络异常等情况，并给出明确toast提示。 | P1 |
| FR-10 | **服务协议查看**：点击协议链接可弹出协议内容详情模态框。 | P2 |

#### **2. API接口定义**

**2.1 发送邮箱验证码**
- **路径**: `/api/v1/auth/send-verification-code`
- **方法**: `POST`
- **请求头**: `Content-Type: application/json`
- **请求参数**:
```json
{
  "email": "string，必填，用户邮箱"
}
```
- **成功响应 (200)**:
```json
{
  "code": 200,
  "message": "验证码发送成功",
  "data": {
    "sid": "string，本次验证会话ID，用于后续校验"
  }
}
```
- **错误响应 (429)**: `{“code”: 429, “message”: “请求过于频繁，请稍后再试”}`
- **错误响应 (400)**: `{“code”: 400, “message”: “邮箱格式错误或已被注册”}`

**2.2 提交注册**
- **路径**: `/api/v1/auth/register`
- **方法**: `POST`
- **请求头**: `Content-Type: application/json`
- **请求参数**:
```json
{
  "email": "string，必填",
  "password": "string，必填，加密后的密码",
  "verificationCode": "string，必填，6位数字",
  "sid": "string，必填，来自发送验证码接口的响应"
}
```
- **成功响应 (201)**:
```json
{
  "code”: 201,
  “message”: “注册成功”，
  “data”: {
    “userId”: “string，系统用户ID”，
    “token”: “string，JWT认证令牌”，
    “walletAddress”: “string，系统为用户创建的OKX Agent钱包地址”，
    “nickname”: “string，默认昵称（邮箱前缀）”
  }
}
```
- **错误响应 (400)**: `{“code”: 400, “message”: “验证码错误或已过期”}`
- **错误响应 (409)**: `{“code”: 409, “message”: “邮箱已被注册”}`

| 字段名 | 类型 | 说明 |

#### **5. 注册流程的完整时序图（文字描述）**

1.  **前端**：用户进入 `/register` 页面，输入邮箱、密码，勾选协议。
2.  **用户**：点击“获取验证码”按钮。
3.  **前端**：校验邮箱格式，若通过，禁用按钮并开始60秒倒计时，调用 **API-2.1** (`/send-verification-code`)。
4.  **后端**：接收请求。
    -   风控校验（频率、IP限制）。
    -   检查邮箱是否已注册。
    -   校验通过后，生成6位随机码和`sid`，记录到`email_verification_codes`表。
    -   调用邮件服务（如SendGrid）发送验证码邮件。
    -   返回`sid`给前端。
5.  **用户**：查看邮箱，获取验证码，并在前端输入验证码。
6.  **用户**：点击“注册”按钮。
7.  **前端**：进行基础格式校验，校验通过后，将密码（前端需先做哈希处理）、邮箱、验证码、`sid`通过 **API-2.2** (`/register`) 发送给后端。
8.  **后端**：接收注册请求。
    -   校验验证码（根据`sid`和`email`查表，核对`code`，检查`used`和`expiresAt`）。
    -   校验通过后，将验证码标记为已使用(`used=true`)。
    -   检查邮箱是否已注册（二次确认）。
    -   **核心步骤**：调用 **OKX Agent Wallet SDK** 的创建钱包接口，生成一个新的钱包地址。
    -   创建用户记录：将邮箱、加密后的密码、默认昵称、**上一步生成的钱包地址** 存入 `users` 表。
    -   生成用户登录 Token (JWT)。
    -   返回成功响应，包含 `userId`, `token`, `walletAddress`, `nickname`。
9.  **前端**：收到成功响应后，将 `token` 和用户信息存入本地状态管理（如 Redux）和持久化存储（如 AsyncStorage）。
10. **前端**：自动导航跳转到 `/welcome` 欢迎引导页面。
11. **异常分支**：
    -   **验证码发送失败**：前端 toast 提示“发送失败，请重试”，按钮恢复可点击状态。
    -   **验证码错误/过期**：后端返回400错误，前端 toast 提示“验证码错误或已过期，请重新获取”。
    -   **邮箱已注册**：后端返回409错误，前端 toast 提示“该邮箱已注册，请直接登录”。
    -   **网络异常**：前端 toast 提示“网络异常，请检查连接”。
    -   **钱包创建失败**：后端应记录详细日志，并返回500错误给前端，提示“系统繁忙，请稍后重试”。
