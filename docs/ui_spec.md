# H Wallet UI 设计规范

## 1. 概述
本文档定义了 H Wallet MVP 版本的 UI 设计规范，涵盖页面、组件、交互与视觉样式。设计严格遵循已批准的 PRD 与 OpenAPI 合约。

## 2. 页面清单与页面目标

| 页面名称 | 页面路径 | 页面目标 | 对应 FR |
| :--- | :--- | :--- | :--- |
| 欢迎/入口页 | `/` | 引导用户进入注册或登录流程，展示产品价值主张。 | - |
| 用户注册页 | `/register` | 引导新用户完成邮箱验证、密码设置，并成功注册账户。 | FR-01, FR-02 |
| 用户登录页 | `/login` | 引导已注册用户使用邮箱和密码登录系统。 | FR-03 |
| 主页（注册/登录后）| `/home` | 展示用户基本信息、钱包状态，作为应用的主要入口。 | FR-05, FR-07 |

**说明**:
- 钱包创建（FR-04）为后端异步流程，无独立页面，其状态在主页展示。
- Token刷新（FR-06）为静默后台行为，无独立页面。
